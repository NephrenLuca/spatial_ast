# 空间语义 AST 设计草稿：面向层次化破坏扩散与 Mamba‑Transformer 混合架构

## 1. 项目定位与设计前提

### 1.1 核心目标
构建一个 **可解释、可编辑、可分层控制** 的 CAD 生成系统。系统以自然语言或工程图纸为输入，通过 **层次化破坏扩散模型**（与 Mamba‑Transformer 混合去噪网络）直接生成 **空间语义 AST**，再确定性编译为 DeepCAD 指令序列，最终在 CAD 内核中执行得到 3D 模型。

### 1.2 设计前提（根据用户选择明确）
- **扩散策略**：采用 **层次化破坏**（Hierarchical Corruption），即在不同扩散时间步 `t` 下，有选择性地破坏 AST 中不同深度的子树。深度越浅（空间粒度越粗）的节点在高噪声阶段被破坏，深度越深（粒度越细）的节点在低噪声阶段被破坏。
- **去噪网络架构**：**Mamba‑Transformer 混合**（层级交替式）。浅层以 Transformer 注意力为主，捕捉局部几何细节；深层以 Mamba 状态空间为主，传播长程结构依赖；并通过跨注意力注入多模态条件。
- **输出表示**：**空间语义 AST**（层次树），其节点类型按空间粒度从 L0（整体）到 L5（参数）组织，树深度严格对应空间包含关系。

### 1.3 整体流程图（扩散 + 混合解码）
```
输入（文本/图纸） → 条件编码器 → 跨注意力注入
        ↓
噪声 AST 序列（Z_T） → 去噪网络（Mamba‑Transformer 混合块 × L 层）
        ↓                          ↓
时间步 t（深度感知调度）      预测噪声 / 掩码
        ↓
去噪后 AST 序列（Z_0） → 约束解码（括号平衡 + 类型检查）→ 合法 AST
        ↓
确定性编译器 → DeepCAD 指令序列 → 执行 → 3D 模型
```

## 2. 空间语义 AST 节点类型（精炼版）

基于第一版草稿，根据层次化破坏的需求进行微调，明确每个节点的 **深度等级（depth level）**，用于噪声调度。

| 等级 | 节点类型 | 子节点约束 | 说明 |
|------|----------|-----------|------|
| **L0** | `Program` | 至少一个 `Solid` | 根节点，代表整个模型 |
| **L1** | `Solid` | 一个 `Sketch` + 零或多个 `Extrude`/`BooleanOp` | 独立实体 |
| **L1** | `Sketch` | 至少一个 `Face` | 2D 草图容器 |
| **L1** | `Extrude` | 无（属性：方向、距离、操作类型） | 拉伸操作 |
| **L1** | `BooleanOp` | 两个 `Solid` | 布尔运算 |
| **L2** | `Face` | 至少一个 `Loop` | 实体的一个面 |
| **L3** | `Loop` | 至少一个 `Edge` | 闭合边界环 |
| **L3** | `Edge` | 一个 `Line`/`Arc`/`Circle` | 几何曲线 |
| **L4** | `Line` | 两个 `Coord` | 线段 |
| **L4** | `Arc` | 三个 `Coord` 或 圆心+半径+角度 | 圆弧 |
| **L4** | `Circle` | 一个 `Coord` + `Radius` | 圆 |
| **L5** | `Coord` | x, y 数值 | 2D 坐标 |
| **L5** | `Distance`, `Angle`, `Radius` | 数值 | 参数 |

**辅助节点**：`List`, `Nil`（用于序列化）。

**序列化格式**：深度优先括号序列，每个节点后跟 `(` + 子节点序列 + `)`。叶子节点直接携带量化后的数值。

## 3. 层次化破坏扩散过程

### 3.1 核心思想
传统的扩散模型（如 DDPM）对序列中的每个 token 独立加噪，忽略了树结构的层次信息。我们提出 **深度感知的层次化破坏**：

- 在训练阶段，对于给定的 AST 树，我们根据节点的深度等级 `d ∈ {0,1,2,3,4,5}` 和当前扩散步 `t` 来决定是否破坏该节点及其子树。
- 破坏操作可以是：① 用掩码 token `[MASK]` 替换该节点；② 将整个子树替换为特殊噪声 token `[NOISE]`；③ 随机打乱子节点顺序（针对 `List` 类型）。

### 3.2 深度感知噪声调度函数
定义破坏概率函数 `p(d, t)`，满足：
- 当 `t` 接近 0（无噪声）时，`p(d, t) ≈ 0`，不破坏任何节点。
- 当 `t` 接近 T（完全噪声）时，`p(d, t)` 对于浅层（小 d）趋近 1，对于深层（大 d）保持较低。
- 中间时刻：`p(d, t)` 随 `d` 增大而减小，即**粗粒度节点先被破坏，细粒度节点后被破坏**。

具体形式（示例）：
\[
p(d, t) = \sigma\left( \frac{t - \tau(d)}{\beta} \right)
\]
其中 `τ(d)` 是深度 `d` 对应的破坏阈值时间（`τ(0) = 0.9T`, `τ(5) = 0.1T`），`σ` 为 sigmoid 函数，`β` 控制陡峭程度。

训练时，对于每个样本，随机采样 `t`，然后遍历 AST 的所有节点，以概率 `p(depth(node), t)` 决定是否破坏该节点及其整棵子树。被破坏的子树全部替换为掩码 token（或噪声 token）。

### 3.3 训练目标
模型接收被破坏后的序列 `X_t`（含掩码）和时间步 `t`，预测被破坏的原始子树（即完成掩码填充）。损失函数可以是：
- 掩码 token 预测的交叉熵（分类到原始 token）。
- 对于数值参数节点，使用 L2 损失回归原始值。

由于我们输出的是 AST 序列（离散 token），最直接的损失是 **masked token prediction cross-entropy**，类似于 BERT 的 MLM，但这里掩码策略是层次化的。

## 4. Mamba‑Transformer 混合去噪网络

### 4.1 整体架构
去噪网络 `ε_θ(X_t, t, cond)` 由 **L 个混合块** 堆叠而成，每个混合块内部采用 **层级交替** 设计：浅层（前 1/3 块）以 Transformer 注意力为主，中层（中间 1/3）混合使用，深层（后 1/3）以 Mamba 为主。同时，在每个块中注入时间步 `t` 和条件嵌入。

**具体设计（参考 Jamba 思路）**：
- 每个混合块包含：
  - **多头注意力（MHA）子层**（使用相对位置编码或 RoPE）
  - **Mamba 子层**（带双向扫描的 SSM）
  - **前馈网络（FFN）**（可选）
  - **跨注意力子层**（用于注入文本/图像条件）
- 顺序：`LayerNorm → MHA → LayerNorm → Mamba → LayerNorm → CrossAttn → FFN`（可调整）

**堆叠规则**：
- 第 1 到 L/3 块：MHA 的维度/头数较大，Mamba 的隐藏维度较小（侧重局部建模）。
- 第 L/3 到 2L/3 块：MHA 和 Mamba 平衡。
- 第 2L/3 到 L 块：MHA 头数减少，Mamba 状态维度增大（侧重长程结构传播）。

### 4.2 深度感知位置编码
为了告知模型每个 token 的深度信息，我们在输入嵌入中加入 **深度嵌入**（depth embedding）：
\[
\mathbf{e}_k = \text{TokenEmbed}(x_k) + \text{PosEmbed}(k) + \text{DepthEmbed}(d_k) + \text{TypeEmbed}(\text{node\_type}_k)
\]
其中 `d_k` 是当前 token 在 AST 中的深度（从 0 开始）。这样模型可以区分不同粒度的 token。

### 4.3 条件注入
文本条件：使用 T5/CLIP 编码器得到序列嵌入，通过 **跨注意力** 注入到每个混合块中。  
图像条件（工程图纸）：使用 ViT/ConvNeXt 编码，同样通过跨注意力注入。  
时间步 `t`：通过 AdaLN（自适应层归一化）或 FiLM 调制每个子层的输出。

### 4.4 约束解码（推理时）
在每个去噪步骤后，对当前序列进行 **约束投影**：
- 括号平衡检查：使用栈扫描，确保括号匹配，不匹配时通过就近闭合或删除非法括号。
- 类型兼容检查：例如 `Coord` 不能直接作为 `Program` 的子节点，必须通过 `Line` 等父节点。可通过预定义语法规则表进行校验和修复。
- 数值范围截断（量化区间 [0,255]）。

最终迭代 `T` 步后，得到合法 AST 序列，再解析为树结构。

## 5. 确定性编译器（AST → DeepCAD 指令序列）

编译器保持与第一版相同的递归下降设计，但需要处理新增的结构控制 token（括号等）在解析时忽略。映射规则示例：

| AST 节点 | 输出指令 |
|---------|----------|
| `Program` | 遍历 `Solid` 列表，顺序输出每个实体的指令 |
| `Solid` | 输出 `begin_solid`（DeepCAD 中无显式，可用 `sketch_start` 作为起始） |
| `Sketch` | 输出 `begin_sketch` + 所有 `Face` 的指令 + `end_sketch` |
| `Face` | 输出 `begin_face` + `Loop` 指令 + `end_face` |
| `Loop` | 输出 `begin_loop` + `Edge` 指令 + `end_loop` |
| `Line` | 输出 `line` 命令，参数为起点和终点的量化坐标 |
| `Extrude` | 输出 `extrude` 命令，参数为距离、方向、操作类型 |

所有参数反量化为原始浮点数（DeepCAD 使用 8‑bit 量化，范围 11‑266，需映射回实际物理尺寸）。

## 6. 训练与评估策略

### 6.1 数据准备
- 从 DeepCAD 数据集（约 178k 个模型）中提取指令序列。
- 通过规则将指令序列转换为空间语义 AST（可编写脚本自动完成）。这会产生 (AST, 指令序列) 配对。
- 若需要自然语言条件，可复用 Text2CAD 或 CAD‑Coder 的文本描述与模型的对应关系。

### 6.2 训练细节
- **模型规模**：使用 4×H100，12‑24 个混合块，隐藏维度 512‑1024，Mamba 状态维度 128‑256。
- **优化器**：AdamW，学习率 1e‑4，梯度裁剪 1.0。
- **扩散步数**：T=1000，使用余弦调度。
- **层次化破坏调度**：`τ(d) = (1 - d/5) * 0.9T`，`β = 0.1T`。
- **损失函数**：掩码预测交叉熵（权重 1.0） + 数值参数回归 L2（权重 0.1）。
- **训练时间**：预计 5‑7 天（4×H100）。

### 6.3 评估指标
- **结构合法性**：生成 AST 中括号匹配率、类型兼容率（越高越好）。
- **编译成功率**：AST → DeepCAD 指令可执行的比例。
- **几何质量**：Chamfer Distance（与原始模型比较），MMD，COV，JSD（同 GeoFusion‑CAD）。
- **可编辑性**：人工评估（随机选取 100 个生成的 AST，用户修改某个参数或子树后重新编译，观察是否得到合理模型）。

## 7. 预期贡献与创新点

1. **首次提出“空间语义 AST”** 用于 DeepCAD 指令序列的层次化表示，显式编码空间粒度（L0‑L5）。
2. **层次化破坏扩散机制**：深度感知的噪声调度，使扩散过程的“粗→细”与 CAD 几何的层次结构严格对齐。
3. **Mamba‑Transformer 混合去噪网络**：层级交替式设计，兼顾局部几何细节（Transformer）与长程结构传播（Mamba）。
4. **端到端可编译、可编辑 CAD 生成框架**：从自然语言/图纸到 AST 再到可执行指令，支持交互式修改。
